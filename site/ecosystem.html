Ecosystem

A native ecosystem — by intent

AbyssBSD does not aim to run the existing Linux desktop applications. That is a deliberate decision, not a missing feature — and it follows from the same reasoning as the rest of the system.

What "existing apps" really are

Let us be plain about it. The modern free-software desktop application is, almost without exception, a Linux UI app: built on GTK, Qt, or Electron; drawing through X11 or Wayland; and wired to D-Bus, polkit, and portal services for everything it cannot do by itself. That is what today's free desktop apps amount to.

To run those applications, an operating system must provide that entire stack. Hosting them on AbyssBSD would mean importing — or re-implementing — X11, Wayland, D-Bus, the portal daemons, the policy services: precisely the accreted layers AbyssBSD was built to refuse.

So AbyssBSD does not. There is no X server, and Wayland compatibility is not a goal. A system cannot both reject that stack and host the applications that require it. AbyssBSD chooses the rejection — with eyes open, and on purpose.

The cost, named honestly

This has a real price, and we will not pretend otherwise: AbyssBSD does not run the GUI applications you use on Linux today, and it is not trying to. This is the BeOS bargain, and we take it deliberately — a coherent system you can hold in your head is worth more than a compatibility layer that drags the whole 40-million-line mess back in through the side door.

An ecosystem built this way starts small. We would rather start small and coherent than large and incoherent.

What does run

The FreeBSD base is kept whole, so the news is not all austere:

  • Every command-line and terminal program runs unchanged — from M1 — compilers, editors, version control, build systems, language runtimes, servers. The FreeBSD base below is untouched.
  • AbyssBSD is a real working system for developers before it has a single graphical app of its own. The terminal emulator is load-bearing by design, and it is the bridge.

Native apps — citizens, not ports

An AbyssBSD app is written for AbyssBSD: a bundle with a capability manifest, message-driven, drawing through the one toolkit. It is born holding no ambient authority and asks for exactly the capabilities it needs — no more.

The result is an application that is coherent with the system instead of fighting it: small, fast, inside the memory and latency budgets, scriptable through the standard interface, and secure by construction. Not a port wearing a costume — a citizen.

The browser — a window to the pre-enshittification web

A desktop needs a browser. A browser does not need to be Chromium.

The path we mean to take is a clean-room, independent engine in the spirit of Ladybird — no Blink, no Gecko, no inherited billion-line dependency — targeting standard, well-specified HTML5 and CSS.

Not the moving target of whatever a surveillance-funded ad company shipped this quarter, but the open web as a documented standard: documents and applications that work because they are built to specifications anyone can read. A browser for the web as it was meant to be — before the tracking, the ad-tech accretion, and the engagement machinery. The pre-enshittification web: standards-first, user-first, and small enough to understand.

A direction, not yet a dated milestone — the roadmap through M5 is the desktop itself. But the browser is the path forward we intend, and it is of a piece with everything else here: refuse the accretion, build the coherent thing.

Built with likeminded folks

An ecosystem is not shipped; it is grown. AbyssBSD's will be grown with people who share the values that produced it — simplicity over accretion, capability security over ambient trust, software that is measured and held to budgets, a desktop and a web you can actually hold in your head.

The application ecosystem is the part of this project that a small team cannot write alone — and it is the part most worth building together. If this is the system you have been waiting for, you are who AbyssBSD is for: as a user, and as a builder.

A return to simplicity — built in the open, in good company.
